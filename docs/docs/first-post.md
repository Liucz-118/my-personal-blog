# 第一篇博客：开源实践作业完成记录
## 一、作业流程
1. 创建公开GitHub仓库存放博客源码
2. 使用MkDocs搭建静态博客，采用Markdown格式撰写文章
3. 配置GitHub Actions工作流，代码提交后自动构建静态网站
4. 开启GitHub Pages实现线上公开访问

## 二、完成过程遇到的问题
编写GitHub Actions部署配置时，YAML文件对缩进空格要求严格，空格错位导致工作流运行失败。核对官方示例，统一缩进空格后成功运行，完成自动部署。

## 三、学习收获
掌握GitHub仓库管理、Markdown写作、CI/CD自动部署、GitHub Pages静态网站发布完整流程。
